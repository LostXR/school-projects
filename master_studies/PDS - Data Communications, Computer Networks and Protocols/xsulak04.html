Description
Project for subject PDS - Data Communications, Computer Networks and Protocols, variant TCPstats. The project is focused on analyzis of TCP communication. The input of this project is a file containing captured network traffic, for example in format 'pcap'. For the sake of simplicity there is always only 1 TCP stream, as from the specification. The aim of this project is to provide a visual interpretation of following information, all of them are taken during the whole connection, for both endpoints:
  • TCP window size
  • Throughput
  • Round-trip time
  • Analysis of sequential numbers
How it works
The solution operates in these steps:
  • firstly, there is a script written in Python. It requires 1 argument, which is the name of the input pcap-like file. As a result, it outputs a log file in Json format, so the result statistical data are saved in Json representation. For parsing the input pcap file I used a Scapy library,
  • secondly, a data from such Json file are displayed in charts on HTML page. It uses Javascript and CSS languages for this purpose. To be more specific, it operates with JQuery, Bootstrap and Highcharts libraries. Metro Dashboard free CSS template has been used for a design part.
Hierarchy of files
  • tcpstats.py - the main script in Python for TCP data extraction. It produces Json file for saving all statistical information,
  • xsulak04.html - this web page. Representation of results in graphs,
  • css/ - folder containing all CSS files including Bootstrap and JQuery. Design is basically one of free CSS templates, to be more specific it is a Metro Dashboard, available on GitHub. There has been implemented just a few features and modifications in it,
  • font/ - folder containing all fonts and few icons in HTML page, all obtained from Metro Dashboard template,
  • img/ - folder containing all images in HTML page,
  • js/ - contains all Javascript files, including JQuery and Highcharts libraries. File myGraphs.js processes Json file and creates all graphs. Actually, it is the only Javascript file, which has been implemented by the author of this work,
  • log/ - contains a file tcpLog.json for storing all necessary values for graphs.
Requirements
  • Python 2.7
  • Scapy library
  • JavaScript
Tips
  • You can perform a zoom in of graphs, you just have to click&drag on the graph area you want to zoom. The opposite, unzoom, can be done afterwards by clicking on button 'Reset zoom'.
  • There are three icons on the top right corner on each category of graphs. The left one is for moving your page upper, on the top level. The middle one is for minimalizing the category and the last icon is for closing the current category. If you reload the page, this will be returned to the default settings.
  • Under these icons you can see another one. It is for printing or exporting of current graph to PNG, JPEG, PDF or SVG.
